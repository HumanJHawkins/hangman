Hangman is a traditional word guessing / spelling game that appears to have originated during the 1800's. The objective is to guess the letters of an obscured word. Correct guesses are shown, whereas incorrect guesses cause parts of a person to be drawn into a gallows. If the word is guessed fully before the man is completed, the game is won. If the person id drawn before the word is guessed, the game is lost.

Guesses can be made by pressing keys on the keyboard while the game window is active, or by selecting (clicking or touching) letters in the letter table on screen. Difficulty settings affect the number of guesses before the person is fully drawn. Word level and word length options filter the words that can be selected.

Good luck!

About Hangman

This implementation of the traditional Hangman game is dedicated to my curious daughter who loves its words, and to my inquiring son who loves to solve these in as few guesses as possible.

It is an example of a single page web app (SPA) built using HTML5, CSS3, and plain Javascript. I've attempted to demonstrate good universal design around color blindness. Universal design is the idea that well designed things should naturally work well for both able-bodied and less able-bodied people. You don't have to sacrifice one for the other. Also, I've attempted to create a concise responsive web design. I'll let you be the judge of whether i've succeeded in these things or not.

Full source code is available at https://github.com/HumanJHawkins/hangman, with the most interesting bits in the .js file there. Copyright ©2018, Jeff Hawkins except as noted in the code.
